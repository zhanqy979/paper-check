# PC-Abbreviation Check

检查 LaTeX 论文正文中的“全称 (缩写)”定义，并分别报告摘要与其他正文。最终表必须合并严格首字母匹配和非严格候选扫描，不得再用独立人工汇总覆盖或补充遗漏项。

## 工作流

1. 确定主 `.tex` 文件。若用户只给目录，优先选择含 `\documentclass` 的文件。
2. 运行：

   ```powershell
   python scripts/check_abbreviations.py <main.tex> -o abbreviation-report.md
   ```

3. 阅读报告并对照原文复核所有大小写提示。专有名词判断具有语义性，不得仅凭脚本结果直接修改论文。
4. 向用户返回报告路径，并简要汇总：

   - 摘要和其他正文各自的定义数量；
   - 重复定义及其 `文件:行号`；
   - 定义后未再出现的缩写；
   - 明显或疑似不规范的全称大小写。

## 强制输出

摘要和其他正文必须各有一张表，表头固定为：

| 序号 | 全称 | 缩写 | 出现次数 |
|---:|---|---|---:|

遵循以下语义：

- 每个定义都单独成行；重复定义不得合并。
- 严格匹配和可信候选定义必须进入同一张最终表，并共同参与计数、重复定义和未使用统计。
- `low-rank adaptation (LoRA)`、`demodulation reference signals (DMRS)` 等非严格首字母对应关系必须作为候选纳入最终表，并另列“非严格缩写定义候选”供人工确认。
- 按定义在对应部分中的出现顺序编号。
- “出现次数”只统计该部分内的非定义位置；定义括号中的缩写不计数。
- `OFDM-based` 等连接形式计为一次 `OFDM` 使用。
- 重复定义统计覆盖摘要与其他正文，并列出每个定义的位置。
- “定义后未再出现”按整个展开后的论文顺序判断。

## 大小写复核

读取 [references/abbreviation-rules.md](references/abbreviation-rules.md) 后处理大小写提示。普通技术术语通常采用句式大小写；人名、地名、机构、标准及其他专名保留规范大写。无法确定时标记“需人工确认”，不要擅自改写。

## 支持范围与限制

- 递归展开 `\input` 和 `\include`，保留源文件与行号。
- 忽略 `%` 注释，并屏蔽常见引用、标签、链接和数学内容。
- 识别全大写、复数和混合大小写括号缩写。
- 对非严格首字母型缩写，要求全称首字母能够按顺序映射到缩写，并限制额外字母数量，降低普通括号文本误报。
- 宏生成的正文、复杂自定义命令、`\includeonly` 和条件编译可能需要人工补查。
- 若脚本没有捕获用户肉眼可见的定义，直接检查原始 TeX，不得声称“全文无问题”。

## 常见错误

- 把摘要中的使用次数加到正文表中。
- 合并重复定义，导致无法定位重复位置。
- 把定义本身算作一次使用。
- 只把候选定义写进人工汇总，却不放入最终缩写表。
- 让自动报告与人工复核分别维护两套重复定义结论。
- 仅凭大写词数量判定专有名词错误。
- 扫描 `.bib`、注释或命令参数并当作正文。
